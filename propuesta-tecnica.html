Propuesta Tecnica - Cliente Vecino Trebol

Esta propuesta separa claramente responsabilidades: el backend propio mantiene clientes y credenciales, y consulta al sistema externo por RUT para traer puntos, compras y ofertas. El frontend recibe un resumen unificado en una sola respuesta.

GET /api/v1/clientes/{rut}/resumen
Mono<Cliente> local = clienteService.findByRut(rut); Mono<ResumenExterno> externo = externalClient.getResumenByRut(rut); return Mono.zip(local, externo) .map(tuple -> resumenMapper.unificar(tuple.getT1(), tuple.getT2())) .timeout(Duration.ofSeconds(3));
Se usa programacion reactiva para ejecutar consultas en paralelo y reducir latencia de carga de la vista.

Contratos API

  • POST /api/v1/auth/login: valida usuario/clave y entrega token JWT.
  • POST /api/v1/clientes: registra nuevo cliente en base local.
  • GET /api/v1/clientes/{rut}/resumen: trae cliente local + datos externos unificados.
  • Seguridad: contrasenas hasheadas (BCrypt/Argon2), rate limit en login.
  • Resiliencia: timeout, retry y circuit breaker para el sistema externo.
  • Fallback: si externo falla, devolver cliente local y estado degradado controlado.
